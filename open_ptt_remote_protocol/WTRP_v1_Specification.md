# WalkieTalkie Remote Protocol (WTRP) v1.0 Specification - FINAL

## 1. Introduction
WTRP v1.0 is a transport-independent binary protocol for Push-To-Talk (PTT) and remote control. This document serves as the final specification for version 1.0.

## 2. Packet Format
WTRP uses a fixed-header binary format.

| Offset | Field | Size (Bytes) | Description |
| :--- | :--- | :--- | :--- |
| 0 | Version | 1 | Protocol version (0x01) |
| 1 | Device ID | 1 | Category of hardware |
| 2 | Command | 1 | Operation identifier |
| 3 | Session ID | 1 | Random ID generated by Peripheral on connection |
| 4 | Seq Number | 1 | Incrementing counter |
| 5 | Payload Len | 1 | Length of payload (N) |
| 6 | Payload | N | Command-specific data |
| 6+N | CRC8 | 1 | SMBUS Polynomial 0x07 |

## 3. Identifiers

### 3.1 Device Category IDs (Byte 1)
| ID | Category |
| :--- | :--- |
| 0x01 | Wear OS Watch |
| 0x02 | Zepp OS Watch |
| 0x03 | Embedded (ESP32/STM32) |
| 0x04 | USB HID / Foot Pedal |
| 0x05 | Bluetooth Remote / Keyfob |
| 0x06 | Android/iOS Companion |
| 0xFF | Custom / Development |

### 3.2 Manufacturer IDs (Reserved in Payload of HELLO)
| ID | Manufacturer |
| :--- | :--- |
| 0x0001 | OpenPTT Reference Hardware |
| 0x0002 | WalkieTalkie Official |
| 0xFFFF | Generic / Open Source |

## 4. Commands (Byte 2)
| Code | Name | Description |
| :--- | :--- | :--- |
| 0x01 | PTT_PRESS | Trigger PTT |
| 0x02 | PTT_RELEASE | Release PTT |
| 0x03 | HELLO | Handshake Start (Peripheral -> Host) |
| 0x04 | HELLO_ACK | Handshake Accept (Host -> Peripheral) |
| 0x05 | READY | Handshake Complete (Peripheral -> Host) |
| 0x06 | HEARTBEAT | Status Update |
| 0x09 | ERROR | Error State |

## 5. Protocol Negotiation (Handshake)
To ensure both ends are ready and session-synced, a 3-way handshake is required.

1. **HELLO**: Sent by Peripheral upon connection.
   - Payload: `[ManufacturerID (2b)][HW_Version (1b)][SW_Version (1b)][AuthChallenge (4b)]`
2. **HELLO_ACK**: Sent by Host.
   - Payload: `[AuthResponse (4b)][CapabilitiesBitmask (2b)]`
3. **READY**: Sent by Peripheral.
   - Peripheral enters operational mode.

## 6. Latency Budgets
Target: < 20ms total propagation.
- **Physical Sensing**: < 2ms (Debouncing)
- **Encoding/Parser**: < 1ms
- **Transport (BLE)**: 7.5ms - 15ms (Connection Interval)
- **Host Processing**: < 2ms

## 7. Security (v1.0)
WTRP v1.0 includes placeholders for simple challenge-response authentication in the HELLO/HELLO_ACK payload. For v1.0, these may be set to 0x00.

## 8. Safety Requirements
- **Heartbeat**: Peripheral SHOULD send HEARTBEAT every 5 seconds if idle.
- **Auto-Release**: Host MUST emit PTT_RELEASE if connection is lost while in PTT_PRESS state.
- **Session Isolation**: Packets with incorrect Session ID MUST be discarded.
